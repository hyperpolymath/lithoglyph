What is FormBase?
[](https://github.com/hyperpolymath/formbase#what-is-formbase)
FormBase is a spreadsheet-database hybrid built on FormDB. Unlike traditional spreadsheet tools, every change is tracked with who made it, when, and why.
Key Features
[](https://github.com/hyperpolymath/formbase#key-features)
FeatureDescription
Provenance by default
Every cell change records who, when, and why - not optional
Full reversibility
Undo any change, forever, with mathematical proof
Cloud-safe
Sync via Dropbox/GDrive without corruption (append-only)
Self-hosted
Your data stays on your servers
PROMPT scores
Research-grade data quality scoring
Multiple views
Grid, Kanban, Calendar, Gallery, Form
Quick Start
[](https://github.com/hyperpolymath/formbase#quick-start)
# Clone the repo
git clone https://github.com/hyperpolymath/formbase
cd formbase

# Start the server (requires Gleam and FormDB)
cd server && gleam run

# Start the UI (requires Deno)
cd ../ui && deno task dev

# Open in browser
open http://localhost:3000

Storage Reality Check

    Not infinite bloat: Only changes are logged, not full copies. A cell edited 10 times stores ~10 small provenance records (who/when/why), not 10 full snapshots.

    Efficient design: Uses CRDTs (Yjs) for conflict-free merges and differential encodingâ€”similar to Git but for spreadsheets. Storage grows linearly with edits, not exponentially.

    â€‹

    Proven approach: Append-only logs power Git, blockchains, and audit systems worldwide without "running out of space" issues.

Provenance Security Benefits

Rather than a risk, this tracking enhances security:

    Tamper-proof: Cryptographic proofs verify history integrity.

    Audit trail: Detects unauthorized changes instantly.

    Time travel: Roll back malicious edits with evidence.

No privacy leaks by defaultâ€”provenance stays local/self-hosted. If concerned, disable cloud sync and run offline-first mode.
