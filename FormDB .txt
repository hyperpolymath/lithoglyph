  Recommended Development Order

  1. formdb - Answer open questions, implement Form.Blocks, Form.Model
  2. fqldt - Once formdb has a query interface to extend
  3. formdb-studio - Once there's data to display
  4. formbase - Once formdb + fqldt work

  What Would Unblock Everything

  The minimum viable formdb needs:

  Form.Blocks  → Append-only storage that works on cloud drives
  Form.Model   → Collections, edges, schema
  Form.Bridge  → Zig FFI so other languages can use it
  Form.Runtime → Basic FQL: INSERT, SELECT, provenance queries

  Once those work, all the other projects become buildable.

  ---
  Bottom line: Focus 100% on formdb until it can:
  1. Store a document
  2. Retrieve it with provenance
  3. Be called from Gleam/ReScript/Julia via FFI
